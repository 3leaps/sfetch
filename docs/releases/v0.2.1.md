# sfetch v0.2.1 Release Notes

## Summary
Self-verify instructions and embedded trust anchors for installed binaries, plus an opt-in real-world corpus runner and expanded checksum pattern coverage.

## Highlights

- **Self-verify & trust anchors**
  - Flags: `--self-verify`, `--show-trust-anchors`, `--show-trust-anchors --json`.
  - Prints deterministic release URLs, expected asset/hash (with offline fallback), platform-specific checksum commands, minisign/PGP commands, embedded minisign pubkey, and a warning that a compromised binary could lie. Dev builds print “no published checksums.”
  - Installer logs: `To verify this installation later: sfetch --self-verify`.
  - Docs updated: README verification section; `docs/security.md` “Verifying Your Installation”.

- **Real-world corpus (opt-in)**
  - Manifest + runner (`scripts/run-corpus.go`, `testdata/corpus.json`, schema `testdata/corpus.schema.json`).
  - Make targets: `make corpus` (fast, dry-run), `make corpus-all` (includes slow), `make corpus-dryrun`, `make corpus-validate` (used by precommit). Default download dir `test-corpus/` (gitignored); override with `CORPUS_DEST`.
  - Docs: `docs/examples.md` corpus section; guidance in `docs/test-corpus/README.md` (includes `GITHUB_TOKEN` usage to avoid rate limits).

- **Checksum discovery expansion**
  - Added version-aware templates and additional defaults (`sha256sum.txt`, `SHA256SUMS_64`, `{{binary}}_{{versionNoPrefix}}_checksums.txt`, etc.) to improve checksum detection in the corpus.

## Usage Notes
- Set `GITHUB_TOKEN` (or `SFETCH_GITHUB_TOKEN`) when running corpus targets to avoid GitHub API rate limits; public-repo read is sufficient.
- Corpus runs are opt-in and dry-run by default.
