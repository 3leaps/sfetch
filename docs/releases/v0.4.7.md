## Summary
Maintenance release: retires the darwin/amd64 (Intel Mac) release target and aligns the installer + release matrix with the narrower supported set. No feature changes, no API changes.

## Highlights

### Intel Mac retirement
- sfetch no longer publishes `sfetch_darwin_amd64.tar.gz` starting v0.4.7.
- `scripts/install-sfetch.sh` exits early with an informative message when it detects `darwin_amd64`, rather than proceeding to a 404 on the missing asset.
- See [ADR-0002](../adr/adr-0002-drop-darwin-amd64.md) for the full rationale — short version: Apple has closed the Intel Mac runway, consumer orgs are on Apple Silicon, and carrying the matrix entry costs more than it returns.

### Supported platforms from v0.4.7
- `darwin_arm64`
- `linux_amd64`, `linux_arm64`
- `windows_amd64`, `windows_arm64`

## Breaking Changes
- **`darwin_amd64` release artifacts no longer exist.** Any automation that downloads `sfetch_darwin_amd64.tar.gz` from `releases/latest/download/` will 404 starting with this release.

## Migration

If you are on Intel Mac, either pin the last supporting release:

```bash
curl -sSfL https://github.com/3leaps/sfetch/releases/download/v0.4.6/install-sfetch.sh | bash -s -- --tag v0.4.6
```

or build from source:

```bash
git clone https://github.com/3leaps/sfetch
cd sfetch
git checkout v0.4.7
GOOS=darwin GOARCH=amd64 go build -o sfetch .
```

Apple Silicon, Linux (amd64/arm64), and Windows (amd64/arm64) users: no action needed.

## Validation
- `go test -race ./...` green
- `make prepush` green
- Installer dry-run from darwin_arm64 succeeds end-to-end against the published release
- Installer dry-run from a simulated darwin_amd64 platform fails with the expected retirement message (no 404 attempt)

## Examples

```bash
# Standard install on a supported platform
curl -sSfL https://github.com/3leaps/sfetch/releases/latest/download/install-sfetch.sh | bash

# Self-update (only applies to supported platforms)
sfetch --self-update --yes
```
