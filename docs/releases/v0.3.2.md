
## Summary
stdout/stderr convention fix and comprehensive test coverage improvements.

## Highlights

### stdout/stderr convention
All human-readable output now goes to stderr; stdout is reserved for JSON only. This enables clean piping in CI/CD pipelines:

```bash
# Extract just the trust score
sfetch --repo foo/bar --dry-run --json 2>/dev/null | jq .trust.score

# Check trust level in scripts
trust=$(sfetch --repo foo/bar --dry-run --json 2>/dev/null | jq -r .trust.levelName)
```

**Affected flags:**
- `--version`, `--version-extended`
- `--self-verify`
- `--show-trust-anchors` (text mode)
- `--dry-run` (success/progress messages)
- `--helpextended`

### Test coverage expansion
Comprehensive test suite added across 5 passes:

| Pass | Focus | Coverage |
|------|-------|----------|
| 1 | Pure functions (inferBinaryName, archiveFormat, tokenCI) | +1.2% |
| 2 | internal/verify edge cases (checksum detection, formats) | +6.6% |
| 3 | CLI flag validation (mutual exclusivity, help/version) | +6.3% |
| 4 | Asset selection (heuristics, match/regex, patterns) | +7.3% |
| 5 | Trust score (algorithms, penalties, score capping) | edge cases |

**Final coverage:** 54.2% main package (up from ~39%)

### shellsentry integration
Added shellsentry to the test corpus with minisign verification fixtures in `testdata/shellsentry/`.

## Breaking Changes
None. The stdout/stderr change is technically a change in output destination but existing workflows using `--json` should be unaffected (JSON still goes to stdout).

## Migration
If you were parsing human-readable output from stdout, update your scripts to read from stderr or use `--json` mode.
