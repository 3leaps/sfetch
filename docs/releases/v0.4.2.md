## Summary
Windows reliability release focused on archive install compatibility and CI coverage.

## Highlights

### Windows install fix
- Added fallback archive binary resolution on Windows so extracted archives can resolve `binaryName.exe` when `binaryName` is requested.
- Fixes Windows install failures seen in downstream bootstrap flows (for example `sfetch` installing `fulmenhq/goneat`).

### CI hardening
- Added Windows dogfood CI coverage that validates `sfetch` can install `goneat` on:
  - `windows-latest` (x64)
  - `windows-latest-arm64-s` (custom arm64 runner)
- Added `actionlint.yaml` custom label configuration so the custom runner label is lint-safe.

## Breaking Changes
None.

## Migration
No migration required. This is a drop-in replacement for v0.4.1.

## Validation
- `go test ./...` passes locally.
- `make precommit` passes locally (tests, lint, schema validation, gosec, cross-platform build).

## Examples

Install goneat on Windows via sfetch:
```bash
sfetch --repo fulmenhq/goneat --tag v0.5.2 --dest-dir .\bin --require-minisign
```

Update sfetch:
```bash
sfetch --self-update --yes
```
