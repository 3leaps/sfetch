## Summary
Proxy support for HTTP(S) downloads in enterprise and locked-down environments.

## Highlights

### Proxy support
sfetch now honors standard proxy environment variables and provides CLI flag overrides for fine-grained control.

**Environment variables** (case-insensitive, zero-config):
- `HTTP_PROXY` / `http_proxy` - proxy for HTTP requests
- `HTTPS_PROXY` / `https_proxy` - proxy for HTTPS requests (used for GitHub API)
- `NO_PROXY` / `no_proxy` - comma-separated list of hosts/domains to bypass

**CLI flags** (override environment when present):
- `--http-proxy <url>` - explicit HTTP proxy URL
- `--https-proxy <url>` - explicit HTTPS proxy URL
- `--no-proxy <csv>` - explicit bypass list

**Examples:**
```bash
# Environment-based (common in CI/enterprise)
export HTTPS_PROXY=http://proxy.corp.example:8080
sfetch --repo 3leaps/sfetch --latest --dest-dir ~/.local/bin

# Flag-based (explicit override)
sfetch --repo 3leaps/sfetch --latest --https-proxy http://localhost:8888 --dest-dir /tmp

# Bypass proxy for specific hosts
sfetch --repo internal/tool --latest --no-proxy "*.internal.corp,localhost" --dest-dir /tmp
```

**Behavior notes:**
- All network fetches (release metadata, assets, keys, checksums) honor proxy settings
- `HTTPS_PROXY` is used for GitHub API calls (always HTTPS)
- `HTTP_PROXY` applies only to plain HTTP requests (rare for GitHub)
- CLI flags take precedence over environment variables
- Uses Go's standard `http.ProxyFromEnvironment` when no flags provided

### Validation coverage
- Table-driven tests for proxy URL parsing and validation
- Environment variable override behavior tests
- `NO_PROXY` host/domain matching tests

## Breaking Changes
None.

## Migration
No migration required. Existing workflows continue to work. To use proxy support, set environment variables or add CLI flags.
